# История разработки «Техно-Золото»

Хронология итеративной разработки сайта-шаблона комиссионного магазина б/у
техники и золота — от локального запуска до боевого деплоя на Beget VPS
с Coolify. Сделано в парном программировании с Claude Code.

> **Безопасность:** все секреты в этом документе заменены на плейсхолдеры
> вида `<TELEGRAM_BOT_TOKEN>`. Реальные значения лежат **только** в Coolify
> Environment Variables и в локальном `.env.local` (исключён из git через
> `.gitignore`). Если вам нужен доступ к боевому окружению — запросите
> отдельно.

---

## Контекст

- **Стартовая точка:** Next.js 14 App Router + TypeScript + Tailwind +
  Supabase + Framer Motion. Локальная разработка, без подключённых
  внешних сервисов.
- **Цель:** довести сайт до боевого состояния и выложить на VPS Beget
  с Coolify под доменом `tehnozoloto.imryazapov.ru`. Проект делается
  как шаблон-демо для дальнейшей продажи клиенту.
- **Заказчик:** Рязапов И. М. — самозанятый (ИНН на 12 цифр).
- **Юридический disclaimer:** для боевого запуска требуется ИП (для б/у
  техники, ОКВЭД 47.79) и спецучёт в Пробирной палате для драгметаллов.
  Самозанятость по 422-ФЗ не подходит для перепродажи и работы с золотом.
  На сайте на время демо стоит баннер «Демонстрационная версия».

## Этап 6. Интеграция платежей (черновик)

- `lib/yookassa.ts` — клиент ЮKassa REST API с поддержкой чека самозанятого
  через «Мой налог». Включается env-переменными `YOOKASSA_SHOP_ID`
  и `YOOKASSA_SECRET_KEY`. Если их нет — `isConfigured()` возвращает false,
  и `/api/orders/create` отдаёт фронту «менеджер свяжется».
- `app/api/orders/create/route.ts` — создание заказа, попытка сохранить
  в Supabase, создание платежа в ЮKassa, возврат `confirmation_url` фронту.
- `app/api/payments/yookassa/webhook/route.ts` — приём уведомлений.
  При `payment.succeeded` помечает заказ `paid` и все его товары
  `is_sold = true` (снятие с витрины).

### Сложность с прокси

Локально dev-машина была за HTTP-прокси (V2Ray на `127.0.0.1:10809`)
для обхода блокировок. Node.js `globalThis.fetch` по дизайну
не использует `HTTPS_PROXY` из env. Решено: установить пакет `undici`
и в `lib/telegram.ts` использовать `undici.fetch` с `ProxyAgent`, если
переменная окружения задана. На прод-сервере без прокси работает обычный
fetch, никаких лишних настроек не требуется.

## Этап 13. Supabase

- Создан прод-проект в регионе Frankfurt.
- Прогнаны миграции `0001_init.sql`, `0003_orders.sql`,
  `0004_test_products.sql` через SQL Editor.
- Получены ключи в **новом формате** Supabase 2025: `sb_publishable_*`
  (заменяет `anon`) и `sb_secret_*` (заменяет `service_role`).
- В Auth → URL Configuration прописан Site URL.

### Проблема: `supabase-js` SDK и новые ключи

Первая попытка использовать `@supabase/supabase-js` 2.103 + новые ключи
не работала — `INSERT` тихо проваливались, `SELECT` возвращал пусто.
В curl-тесте с тем же ключом прямой REST вызов работал (HTTP 201/200).

**Решение:** написан `lib/supabase/rest.ts` — прямой REST-клиент
через `fetch`, минующий SDK. Используется во всех серверных операциях
(INSERT в `leads`, `orders`, UPDATE в `orders`/`products`, SELECT в
админке). Гарантированно работает с любым форматом ключа.

### Дополнительная заминка

Build-time переменные `NEXT_PUBLIC_SUPABASE_URL` и
`NEXT_PUBLIC_SUPABASE_ANON_KEY` в Coolify первоначально не сохранились —
в UI нужно нажимать **Update** **под каждой конкретной строкой**, не
общий Save. После этого пришлось пересобрать Docker-образ через **Deploy**
(не Restart — переменные зашиваются на этапе билда).

   - Сменить `ADMIN_PASSWORD` с тестового на боевой.
   - Сделать `SUPABASE_SERVICE_ROLE_KEY` rotate, если он когда-либо

## Известные технические долги

- В нескольких местах используется `as never` для обхода багов
  типизации `@supabase/ssr` 0.10 + новых ключей. После решения проблемы
  Supabase это можно убрать (заодно убрать пакет `undici` если прокси
  на сервере не нужен).
- Для приёма заявок и заказов из Bitrix24 есть Edge Function
  `supabase/functions/sync-bitrix/index.ts`, но она пока не активирована
  на проде — Database Webhook не настроен.
- В админке нет пагинации; при росте заказов 50+ нужно добавить.
- IP-whitelist для webhook ЮKassa не реализован — добавить перед прод-запуском.

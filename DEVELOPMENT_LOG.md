# История разработки «Техно-Золото»

Хронология итеративной разработки сайта-шаблона комиссионного магазина б/у
техники и золота — от локального запуска до боевого деплоя на Beget VPS
с Coolify. Сделано в парном программировании с Claude Code.

> **Безопасность:** все секреты в этом документе заменены на плейсхолдеры
> вида `<TELEGRAM_BOT_TOKEN>`. Реальные значения лежат **только** в Coolify
> Environment Variables и в локальном `.env.local` (исключён из git через
> `.gitignore`). Если вам нужен доступ к боевому окружению — запросите
> отдельно.

---

## Контекст

- **Стартовая точка:** Next.js 14 App Router + TypeScript + Tailwind +
  Supabase + Framer Motion. Локальная разработка, без подключённых
  внешних сервисов.
- **Цель:** довести сайт до боевого состояния и выложить на VPS Beget
  с Coolify под доменом `tehnozoloto.imryazapov.ru`. Проект делается
  как шаблон-демо для дальнейшей продажи клиенту.
- **Заказчик:** Рязапов И. М. — самозанятый (ИНН на 12 цифр).
- **Юридический disclaimer:** для боевого запуска требуется ИП (для б/у
  техники, ОКВЭД 47.79) и спецучёт в Пробирной палате для драгметаллов.
  Самозанятость по 422-ФЗ не подходит для перепродажи и работы с золотом.
  На сайте на время демо стоит баннер «Демонстрационная версия».

### Сложность с прокси

Локально dev-машина была за HTTP-прокси (V2Ray на `127.0.0.1:10809`)
для обхода блокировок. Node.js `globalThis.fetch` по дизайну
не использует `HTTPS_PROXY` из env. Решено: установить пакет `undici`
и в `lib/telegram.ts` использовать `undici.fetch` с `ProxyAgent`, если
переменная окружения задана. На прод-сервере без прокси работает обычный
fetch, никаких лишних настроек не требуется.

## Этап 13. Supabase

- Создан прод-проект в регионе Frankfurt.
- Прогнаны миграции `0001_init.sql`, `0003_orders.sql`,
  `0004_test_products.sql` через SQL Editor.
- Получены ключи в **новом формате** Supabase 2025: `sb_publishable_*`
  (заменяет `anon`) и `sb_secret_*` (заменяет `service_role`).
- В Auth → URL Configuration прописан Site URL.

### Проблема: `supabase-js` SDK и новые ключи

Первая попытка использовать `@supabase/supabase-js` 2.103 + новые ключи
не работала — `INSERT` тихо проваливались, `SELECT` возвращал пусто.
В curl-тесте с тем же ключом прямой REST вызов работал (HTTP 201/200).

**Решение:** написан `lib/supabase/rest.ts` — прямой REST-клиент
через `fetch`, минующий SDK. Используется во всех серверных операциях
(INSERT в `leads`, `orders`, UPDATE в `orders`/`products`, SELECT в
админке). Гарантированно работает с любым форматом ключа.

### Дополнительная заминка

Build-time переменные `NEXT_PUBLIC_SUPABASE_URL` и
`NEXT_PUBLIC_SUPABASE_ANON_KEY` в Coolify первоначально не сохранились —
в UI нужно нажимать **Update** **под каждой конкретной строкой**, не
общий Save. После этого пришлось пересобрать Docker-образ через **Deploy**
(не Restart — переменные зашиваются на этапе билда).

## Этап 15. Каталог наполнен ассортиментом

Каталог расширен с 11 до 64 товаров с реальными стоковыми фотографиями.
Реализовано через два scripts/-генератора, выдающих SQL-миграции:

- `scripts/generate-seed.mjs` → `supabase/migrations/0005_full_seed.sql` —
  DELETE старых mock-товаров + INSERT 54 позиций из `lib/products-data.ts`
  плюс 5 ювелирных, с placeholder-картинками `placehold.co` категорийных
  цветов (серый для смартфонов, тёмный для ноутбуков, золотой для золота).
- `scripts/generate-images-update.mjs` →
  `supabase/migrations/0006_unsplash_images.sql` — 46 `UPDATE`-запросов
  по `title`, ставящих Unsplash-фото из старого mock. UUID товаров
  не меняются, ювелирка и тестовые товары не трогаются (у них остаются
  placehold.co — выглядит уместно для шаблона).

Итог в БД: 53 позиции с Unsplash-фото + 5 ювелирных с золотистым
placeholder'ом + 5 тестовых с красным = 63 видимых товара в каталоге.

## Известные технические долги

- В нескольких местах используется `as never` для обхода багов
  типизации `@supabase/ssr` 0.10 + новых ключей. После решения проблемы
  Supabase это можно убрать (заодно убрать пакет `undici` если прокси
  на сервере не нужен).
- Для приёма заявок и заказов из Bitrix24 есть Edge Function
  `supabase/functions/sync-bitrix/index.ts`, но она пока не активирована
  на проде — Database Webhook не настроен.
- В админке нет пагинации; при росте заказов 50+ нужно добавить.
- IP-whitelist для webhook ЮKassa не реализован — добавить перед прод-запуском.
- AI-автозаполнение (`lib/ai-autofill.ts`) использует обычный fetch без
  прокси-обёртки. На прод-сервере в РФ запросы к Anthropic/OpenAI режутся
  геоблоком. Решения: прокси-обёртка через undici или переход на
  proxyapi.ru / YandexGPT / GigaChat.

## Релиз v2.0 — «полный CRUD товаров с админкой» (2026-05-24)

Большой скачок от v1.0. Что добавлено за период между v1.0 и v2.0

### Этап 16. Полная админка управления товарами

Появилась настоящая админ-форма создания и редактирования товара —
больше не нужно править preset/migrations в коде. Что включает форма
(`app/admin/products/`):

- **Список товаров** `/admin/products` с фильтрами по статусу
  (все / опубликованные / черновики / проданные), поиском по названию,
  кнопками «Редактировать» и «Удалить» (soft-delete через `is_sold +
  is_published`).
- **Форма создания/редактирования** `ProductForm.tsx` со всеми блоками:
  - **Title autocomplete** — при наборе подсказывает модели из preset
    с фильтрацией по выбранной категории и бренду.
  - **BrandPicker** — топ-10 брендов чипсами для категории + спойлер
    «Другие бренды» с полным списком + кнопка «Указать свой бренд».
  - **Drag-and-drop загрузка фото** с автоматическим ресайзом до
    1920×1920 px и сжатием до JPEG quality 88. Прямая загрузка в
    Supabase Storage из браузера (минуя сервер). До 10 фото, drag для
    смены порядка, первое = обложка.
  - **Комплектность** с пресет-чипсами по 35 категориям
    (`lib/preset-complectation.ts`) + ручное добавление.
  - **Характеристики** — свободные пары «параметр → значение» + парсер
    из текста.
  - **Battery state** — свободная строка (85%, или 500 циклов, или
    «10000 mAh из 19000»).
  - Чекбокс **«Опубликовать»** — товар в черновиках не виден в каталоге.

### Этап 17. Парсер характеристик из текста

`lib/parse-specs.ts` + `SpecsParserModal.tsx`. Менеджер копирует блок
характеристик с DNS/Авито/M.Video/Озон/Яндекс.Маркет, парсер
автоматически раскладывает на пары. Поддерживает 5 форматов
(построчный, inline в одной строке, через таб, тире, двоеточие).
Чёрный список из ~100 слов (бренды, единицы, термины) исключает
ложные разрезы типа «Видеокарта = UHD / Graphics = ...». Точность
85-95% на типовых сайтах, остальное правится в превью модалки.

### Этап 18. AI-автозаполнение

`lib/ai-autofill.ts` + `app/api/admin/ai-autofill/route.ts`. Кнопка
**✨ Распознать AI** над полем «Название». Менеджер вводит любую
модель («ASUS X550J 8GB 1TB»), AI возвращает title / brand / category
/ description / specs. Поддерживается Anthropic Claude (приоритет)
и OpenAI как fallback. Активируется env-переменными `ANTHROPIC_API_KEY`
или `OPENAI_API_KEY`. Без ключей кнопка показывает ошибку.
